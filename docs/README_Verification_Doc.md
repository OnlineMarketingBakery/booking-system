# Google OAuth Verification – Document for Owner

## File

**`Google_OAuth_Verification_Readiness.docx.html`**

This file contains:
- A **checklist** the owner can use to mark what is done and what is remaining before submitting for Google OAuth verification.
- The full **verification readiness details** (what’s OK, what to fix, Privacy Policy, domains, HTTPS, demo video, etc.).

## How to get a .docx file to send to the owner

1. **Open the file**  
   Double‑click `Google_OAuth_Verification_Readiness.docx.html` (it will open in your browser or in Microsoft Word, depending on your system).

2. **If it opened in a browser:**  
   - Right‑click → “Open with” → **Microsoft Word**, or  
   - Open Word → File → Open → select `Google_OAuth_Verification_Readiness.docx.html`.

3. **Save as Word document**  
   In Word: **File → Save As** → choose **Word Document (*.docx)** → save (e.g. as `Google_OAuth_Verification_Checklist.docx`).

4. **Send the .docx** to the owner so they can use the checklist and see what’s done vs remaining.

## Using the checklist

The checklist at the top has 7 items. The owner can:
- **In Word:** Replace ☐ with ☑ (or use a checkbox) when an item is done, or add a “Done” column and tick there.
- **Print:** Print the document and tick items on paper.

When all items are done, the app is ready to submit for verification.
